Story behind the features

Odoo behavior : Account receivable and account payable are mandatory fields on the partner form view, which can lock you in several situation for an info which is 95% of time the same for all the customer.

Falinwa behavior : as for other accounting properties, Falinwa standard add the notion of default account receivable, defaut account payable on the Accounting configuration panel and remove the mandatory property of Account receivable and account payable on the partner. As for other fields, Odoo will select the partner value if it's set and will select the default value if the partner field is empty.

Module + : the responsible of the partner database does no more have to care about the accounting property of the partners. And if some partners have specific properties the accountant can adjust the configuration. It's much more convenient to use.

Business case : With Odoo when you add a new entity to your database, the fields Account payable and Account receivable, which are local (ie different for each company) are empty. So you need to make a mass export - import before working on the existing partners. Thanks to the module, this problem does not exist anymore.

Falinwa recommendation :

  • Installation : 100% of the database.
  • Recommandation / Best practice : as soon as the accounting apps is set-up, all the default value have to be set-up to ensure that the system is properly working.
