Story behind the features

Odoo behavior : Odoo does not propose a generic product organization in order to start with Odoo. You have to do it from scratch and without model, which often implies something quite messy that you try to clean after 1 year.

Falinwa behavior : we propose a generic framework with a default category structure and generic products. Purpose is to add the notion of Opportunity expected amount in Group currency in order to be able to make report for each entity or for the group, both on the opportunity amount and on the expected amount.

Module + : based on the 10 years Falinwa experience as outsourced CFO and master data management.

  • WARNING: master data is not "done" by the ERP but by the final user, ie you. And the most challenging topic at almost all the integration is to import a clean master data into the system. Do not undersestimate the effort required for that. It's often as long and as expensive than the integration itself.

Falinwa recommendation :

  • Installation : Falinwa install this module on all the customers because they did not have a mature product database organization.
  • Recommendation / Best practice : if the generic structure does not match your needs, Falinwa recommends to take some consulting hours in order to design a Taylor-made product category structure adapted to your business. Once the architecture is done, you will be able to create the new categories, the new products in a robust way.
