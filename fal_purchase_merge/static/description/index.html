Story behind the features

Odoo behavior : Odoo removed the possibility to merge RFQ and automatically do it when required.

Falinwa behavior : you have the possibility under some constraints to merge several RFQ into one in order to organize your purchase order as you wish.

Module + : purchaser control how he wants to manage the order with his suppliers.

Falinwa recommendation :

  • Installation : 100% of the database.
  • Recommendation / Best practice : to be combined with the procurement request module and ensure good task segregation between the one who request and the purchaser who select Supplier + Price.

To merge draft Purchase order, there are some condition required :

  • Draft PO must have same vendor
  • Draft PO must have same currency
  • Draft PO must have same company
