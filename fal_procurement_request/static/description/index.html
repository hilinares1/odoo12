Story behind the features

Odoo behavior : to request some products, Odoo directly proposes to make a purchase order or a call to tender - which implies to know the supplier and to have purchase access rights.

Falinwa behavior : you can make a procurement request on a product without making a RFQ which implies information on supplier and price. You only requires a product and a quantity. Then, the purchase department can order to the supplier based on this information.

Module + : a must to split the notion of product requirement from RFQ for all the companies, and especially for the factories, where the workshop people knows the need (ie product + quantity) and the purchase department knows the conditions (ie supplier & price).

Businsess case : the workshop manager needs some tools for his workshop. He directly makes a procurement request in the system. Then the purchase department can make the order to the supplier.

Falinwa recommendation :

  • Installation: all the companies which make advanced purchases.
  • Recommendation / Best practice: with this module, all the procurement request - even of office stuff - can be done by any employee (with enough access right) to the purchase department. Like that, you can track the origin of a demand for any purchase which is a key info in terms of traceability.

Why you need module ?

On Odoo default, when a user ask for a product, he/she create a Request for Quotation (RFQ) even for non purchase staff, which means other employee need to have a purchaser access right, which is not correct. with this modules, employee can make a procurement request through the menu or directly from the product.
