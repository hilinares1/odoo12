Story behind the features

Odoo behavior : Odoo does not propose option to force records before a date. Collaborators are reminded but not locked.

Falinwa behavior : you can define a date for filling your timesheet - mission expenses for all the collaborators and a date to validate them by their respective N+1, This option gives the possibility to uniformize the internal behavior.

Module + : the notion of locking period is the only way to ensure your closing including all the timesheets of your collaborators.

Falinwa recommendation :

  • Installation : advanced HR project with strong controlling policy.
  • Recommendation / Best practice: install for the module after a few months using timesheet to avoid any side effect - demotivation. Time to take the habit of the internal process.

    • take the habit to never put any records on a parent, but organize your master data in order to record only on children. Then, when you call the hierarchy view, combine it with a filter on regular in order to get only the general tree.
